# PTMK Document Service

Go service for storing, reading, listing, and deleting user documents.
API contract is described in `api.yaml`, server/client code is generated by `ogen` into `pkg/api`.

## Stack

- Go `1.25`
- PostgreSQL `16`
- HTTP server generated by `ogen` (OpenAPI 3.0)
- SQL builder: `github.com/Masterminds/squirrel`
- DB driver/pool: `pgx/v5`

## Project Structure

- `cmd/` - application entrypoint (`main.go`)
- `internal/app` - HTTP handlers
- `internal/service` - application/service layer
- `internal/repository` - SQL queries and transactions
- `internal/model` - domain models and shared errors
- `migrations` - SQL migrations
- `pkg/api` - generated OpenAPI code
- `api.yaml` - source OpenAPI spec

## Run With Docker Compose

1. Start all services:

```bash
docker compose up --build
```

2. Service will be available at:

```text
http://localhost:8080
```

Important details:
- Postgres is started as `ptmk-postgres`.
- Migrations are mounted to `/docker-entrypoint-initdb.d` and run automatically only on first DB initialization (empty volume).
- If migration files changed and volume already exists, recreate DB volume:

```bash
docker compose down -v
docker compose up --build
```

## Run Locally (without app container)

1. Start only Postgres:

```bash
docker compose up -d postgres
```

2. Export `DATABASE_URL`:

```bash
export DATABASE_URL='postgres://ptmk:123@localhost:5432/db?sslmode=disable'
```

3. Run service:

```bash
go run ./cmd/app
```

By default, if `DATABASE_URL` is not set, app uses:

```text
postgres://ptmk:123@localhost:5432/db?sslmode=disable
```

## API Overview

Base URL: `http://localhost:8080`

### Endpoints

- `POST /documents` - create document (`201`, `400`, `500`)
- `GET /documents` - list user documents (`200`, `400`, `500`)
- `GET /documents/{id}` - get document by ID (`200`, `404`, `500`)
- `DELETE /documents/by-number/{document_number}` - delete by document number (`204`, `404`, `500`)

OpenAPI spec: [`api.yaml`](api.yaml)

## Request/Response Notes

- Dates use OpenAPI `format: date` (`YYYY-MM-DD`).
- `persons[].type` supports:
  - `LEGAL_ENTITY`
  - `NATURAL_PERSON`
- For listing, service validates `sort` against strict allow-list:
  - `company_name`
  - `first_name`
  - `middle_name`
  - `last_name`
  - `initials`
  - `start_date`
  - `expire_date`

## HTTP Request Templates

Use these as reusable templates (replace placeholders in `<...>`):

```bash
BASE_URL="http://localhost:8080"
```

### `POST /documents`

```bash
curl -X POST "$BASE_URL/documents" \
  -H 'Content-Type: application/json' \
  -d '{
    "user_id": <USER_ID>,
    "document_type": "<DOCUMENT_TYPE>",
    "document_number": "<DOCUMENT_NUMBER>",
    "date_created": "<YYYY-MM-DD>",
    "date_end": null,
    "persons": [
      {
        "type": "LEGAL_ENTITY",
        "legal_entity": { "name": "<COMPANY_NAME>" }
      },
      {
        "type": "NATURAL_PERSON",
        "natural_person": {
          "first_name": "<FIRST_NAME>",
          "middle_name": null,
          "last_name": "<LAST_NAME>",
          "initials": null
        }
      }
    ]
  }'
```

### `GET /documents`

```bash
curl "$BASE_URL/documents?user_id=<USER_ID>&sort=<SORT_FIELD>&limit=<LIMIT>"
```

### `GET /documents/{id}`

```bash
curl "$BASE_URL/documents/<DOCUMENT_ID>"
```

### `DELETE /documents/by-number/{document_number}`

```bash
curl -X DELETE "$BASE_URL/documents/by-number/<DOCUMENT_NUMBER>" -i
```

## Example Requests

### Create document

```bash
curl -X POST http://localhost:8080/documents \
  -H 'Content-Type: application/json' \
  -d '{
    "user_id": 1,
    "document_type": "passport",
    "document_number": "AB-123456",
    "date_created": "2026-03-01",
    "date_end": null,
    "persons": [
      {
        "type": "LEGAL_ENTITY",
        "legal_entity": { "name": "Acme LLC" }
      },
      {
        "type": "NATURAL_PERSON",
        "natural_person": {
          "first_name": "Ivan",
          "last_name": "Ivanov"
        }
      }
    ]
  }'
```

### List documents

```bash
curl 'http://localhost:8080/documents?user_id=1&sort=start_date&limit=20'
```

### Get document by ID

```bash
curl 'http://localhost:8080/documents/1'
```

### Delete by document number

```bash
curl -X DELETE 'http://localhost:8080/documents/by-number/AB-123456' -i
```

## Database

Main tables:
- `users`
- `documents`
- `doc_parties`

Current important migration details:
- `documents.doc_number` column exists.
- `doc_parties.doc_id -> documents.id` foreign key has `ON DELETE CASCADE` (migration `20260307170024_added_delete_on_cascade.sql`).

Note:
- `doc_number` is currently not unique on DB level, so delete-by-number may affect multiple rows if duplicates exist.
